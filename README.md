# NLM — AI Visibility for Small Businesses

MBA AI Ventures class project.

NLM helps small businesses (gyms, restaurants, boutique stores) understand and improve how AI assistants describe them. When a potential customer asks ChatGPT or Gemini "best yoga studio in Shoreditch", does your business come up? NLM answers that question and tells you what to do about it and helps you implement it.

## What it does

**Step 1 — Extract**: Scrapes the business's website and Google Business Profile to build a structured profile (name, type, services, location, observable signals like Schema markup, reviews, FAQs, social links).

**Step 2 — Score**: Generates 12 realistic customer queries across 9 intent categories, sends each query to three LLMs (ChatGPT, Claude, Gemini) with live web search enabled, and detects whether the business is mentioned. Score = mentions / total queries per LLM; overall score = average across LLMs.

**Step 3 — Recommend + Act**: Produces gap-based recommendations ordered by impact (High/Medium/Low), grounded only in signals the business is actually missing. Also generates ready-to-use content: a blog post draft, FAQ page, Schema.org markup snippet, optimised meta description, and review response templates.

## System architecture

```
Browser
  |
  | POST /api/analyze   → scrape URL → extract BusinessProfile via GPT-4o-mini
  |                       → enrich via Google Places API (GBP, reviews, hours)
  |
  | POST /api/score     → generate 12 intents (9 GEO buckets) → 12 queries
  |                       → query OpenAI + Anthropic + Gemini in parallel
  |                       → fuzzy mention detection → per-LLM scores + summary
  |
  | POST /api/recommend → gap analysis from BusinessProfile signals
  |                       → ordered recommendations (High/Medium/Low impact)
  |
  | POST /api/actions   → action cards from signal gaps
  |                       → blog post + FAQ generated by GPT-4o-mini
  |                       → template fallback if LLM call fails
```

Everything runs inside Next.js API routes — no separate backend.

## Running locally

### Prerequisites

- Node.js 18+
- API keys for: OpenAI, Anthropic, Google Gemini, Google Places

### Setup

```bash
git clone <repo>
cd nlm/frontend
npm install
```

Create `frontend/.env.local`:

```
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
GEMINI_API_KEY=...
GOOGLE_PLACES_API_KEY=...
```

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## API routes

### `POST /api/analyze`

Scrapes a business URL and returns a `BusinessProfile`.

**Body**: `{ url: string }`

**Returns**: `BusinessProfile` — name, type, location, description, services list, and observed signals (Schema markup, blog, FAQ, social links, GBP status, review count/rating, photo count, opening hours).

### `POST /api/score`

Generates queries and scores LLM visibility.

**Body**: `{ profile: BusinessProfile, queryCount?: number }`

- `queryCount`: 1–12, default 12. Controls how many queries are sent to LLMs (useful for fast testing).

**Returns**: `ScoreResult` — per-LLM scores, mention counts, generated intents and queries, a plain-English summary per intent bucket, and the full (query × LLM) debug table.

### `POST /api/recommend`

Produces gap-based recommendations.

**Body**: `{ profile: BusinessProfile }`

**Returns**: `Recommendation[]` — each with category, impact level (High/Medium/Low), explanation, and suggested action.

### `POST /api/actions`

Generates ready-to-use content and action cards.

**Body**: `{ profile: BusinessProfile }`

**Returns**: `ActionCard[]` — includes LLM-generated blog post, FAQ, and Schema snippet, plus templated cards for meta description, title tag, Maps embed, social bio, and review response templates.

## Key design decisions

**Score definition**: mentions / total queries. Simple, explainable, and directly tied to real user behaviour.

**Query generation**: Two-step process — first generate 12 customer *intents* covering 9 GEO buckets (Discovery, Fit/Persona, Constraints, Quality/Trust, Experience/Vibe, Price/Value, Comparison, Logistics, Problem-based), then convert each intent into a natural search query. Queries never include the business name — it's a pure discovery test.

**Mention detection**: Fuzzy matching strips business-type suffixes (e.g. "Gym", "Restaurant"), checks domain stems, and tests multi-token brand combinations. Case-insensitive.

**Anthropic rate limits**: Claude API calls are batched 3-at-a-time with a 2-second delay and exponential backoff on 429s. OpenAI and Gemini run fully in parallel.

**Recommendations grounded in signals**: The recommendation engine only flags things the business is actually missing. It never suggests "add reviews" if the business already has reviews.

## Known limitations

- **Pipeline latency**: The full pipeline takes 30–60 seconds. Anthropic calls are the bottleneck (rate limits). A Vercel Hobby deployment has a ~60s function timeout — close to the limit.
- **No caching**: Each submission re-runs the full pipeline. Repeated demos burn API credits.
- **Error handling**: Partial failures (e.g. one LLM timing out) are not yet handled gracefully — the full request will fail.
- **Review analysis**: Individual review text analysis (sentiment/theme extraction) is not yet implemented.
- **Improve tier**: Autonomous content actions (auto-publish blog posts, respond to reviews) are product vision, not implemented.
